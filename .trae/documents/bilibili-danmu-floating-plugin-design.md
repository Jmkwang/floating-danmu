# B站弹幕悬浮插件 - 设计方案

## 一、项目概述

**项目名称**: Floating Danmu

**核心功能**: 一个浏览器插件 + 桌面客户端，让用户在观看B站直播或视频时，可以通过桌面悬浮窗口直接发送弹幕/评论，浏览器可以最小化。

**目标用户**: 习惯边听直播/视频边做其他事情的用户，需要快速发弹幕但不想切换到浏览器窗口。

## 三、组件设计

### 3.1 浏览器插件 (Manifest V3)

**核心功能**:
1. **B站页面检测**: 监听tab变化，检测当前页面是否为B站直播间或视频页
2. **登录状态读取**: 从页面DOM/cookie中提取B站登录凭证
3. **悬浮UI**: 在页面上显示mini输入面板（可选，悬浮窗口也能用）
4. **通信**: 与桌面客户端通信，传递弹幕内容和登录凭证

**检测页面类型**:
| 页面类型 | URL特征 | 支持功能 |
|---------|---------|---------|
| 直播间 | `live.bilibili.com/*` | 发弹幕 |
| 视频页 | `bilibili.com/video/*` | 发弹幕 + 发评论 |

### 3.2 桌面客户端 (Electron)

**核心功能**:
1. **系统托盘**: 最小化到托盘，后台运行，不占用任务栏
2. **悬浮窗口**: 置顶、小尺寸、可拖拽位置
3. **弹幕发送**: 接收插件消息，调用B站API发送弹幕
4. **状态显示**: 显示当前B站页面信息（直播间/视频标题）

**窗口规格**:
- 尺寸: 320px × 160px
- 位置: 右下角
- 样式: 毛玻璃背景、圆角、现代简约

## 四、数据流

### 4.1 发送弹幕流程
```
用户在悬浮窗口输入弹幕
         │
         ▼
浏览器插件读取当前B站页面的登录Cookie
         │
         ▼
插件将【弹幕内容 + Cookie + 页面类型】发送给桌面客户端
         │
         ▼
桌面客户端调用B站API发送弹幕
         │
         ▼
返回发送结果（成功/失败）给用户
```

### 4.2 API调用

#### 直播间弹幕
```
POST https://api.live.bilibili.com/msg/send
Headers: Cookie: SESSDATA=xxx; bili_jct=xxx
Body: roomid=xxx&msg=弹幕内容&csrf=xxx
```

#### 视频弹幕
```
POST https://api.bilibili.com/x/v2/reply/add
Headers: Cookie: SESSDATA=xxx; bili_jct=xxx
Body: oid=视频avid&type=1&message=弹幕内容&csrf=xxx
```

## 五、功能清单

## 七、验收标准

1. 安装插件和客户端后，能自动检测B站直播间/视频页面
2. 悬浮窗口显示当前页面信息
3. 用户可在悬浮窗口输入并发送弹幕，弹幕成功出现在直播间/视频中
4. 客户端可最小化到系统托盘，后台运行
5. 发送结果有明确的成功/失败反馈
